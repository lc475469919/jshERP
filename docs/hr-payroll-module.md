# 员工、考勤与工资模块设计

## 目标

在单公司 ERP 中补齐员工管理、打卡考勤、计时工资、计件工资和工资发放能力，并与生产、审批、财务模块形成闭环。

## 模块边界

### 员工档案

- 员工编号、姓名、手机号、部门、岗位
- 入职日期、离职日期、在职状态
- 基础工资、工资类型
- 小程序登录用户关联

### 考勤管理

- 班次设置
- 上下班打卡
- 外勤打卡
- 请假、加班、调休
- 迟到、早退、缺勤统计
- 考勤月报

### 计件工资

- 工序计件单价
- 按生产任务、工序、员工记录良品数、不良数、报废数
- 来源优先复用工序汇报
- 支持质检扣减规则
- 生成员工月度计件明细

### 工资管理

- 工资项目
- 月度工资单
- 基本工资
- 计时工资
- 计件工资
- 加班工资
- 请假/缺勤扣款
- 其他补贴/扣款
- 实发工资
- 工资发放记录

## 业务闭环

1. 员工在小程序或后台维护档案。
2. 员工通过小程序打卡，形成考勤记录。
3. 生产员工提交工序汇报，形成计件来源。
4. 月末按员工生成考勤汇总和计件汇总。
5. 系统生成工资单，财务确认后发放。
6. 员工在小程序查看工资条。

## 和现有模块的关系

- 生产模块：工序汇报生成计件工资来源。
- 审批模块：请假、加班、工资单确认可走审批。
- 财务模块：工资单确认后可生成付款或工资发放记录。
- 小程序：打卡、请假/加班申请、查看考勤、查看工资条。

## 菜单规划

后台菜单：

- 人事工资
- 员工档案
- 班次设置
- 打卡记录
- 请假加班
- 考勤月报
- 计件工资
- 工资单
- 工资发放

小程序入口：

- 打卡
- 我的考勤
- 请假/加班
- 我的工资条
- 工序汇报

## 开发顺序

### 第 1 步：员工档案

- 后端员工表和基础 API
- 后台员工列表/编辑
- 小程序员工资料查看

### 第 2 步：打卡考勤

- 班次设置
- 打卡记录
- 小程序上下班打卡
- 考勤月报初版

### 第 3 步：计件工资

- 工序单价
- 工序汇报关联员工
- 计件明细统计
- 按月汇总

### 第 4 步：工资单

- 工资项目
- 月度工资单生成
- 工资确认/锁定
- 工资发放记录

### 第 5 步：审批和财务联动

- 请假/加班审批
- 工资单审批
- 工资发放生成财务记录

## 关键规则

- 工资单生成后，来源考勤和计件数据仍可调整。
- 工资单确认后，来源数据锁定，不允许随意修改。
- 工资单发放后，只能通过冲销或补发处理差异。
- 计件工资以工序汇报为主要来源，后台允许手工补录调整。
- 小程序打卡先支持单公司固定地点，后续再扩展多地点和外勤范围。
