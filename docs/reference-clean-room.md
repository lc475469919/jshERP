# 参考系统 clean-room 规则

## 可以参考

- 菜单结构
- 页面布局
- 列表筛选方式
- 表格列组织方式
- 表单分组方式
- 按钮和操作入口位置
- 业务流程
- 字段含义
- 单据状态
- 计算公式
- 报表口径
- 常见操作入口
- 用户使用路径

## 不直接复制

- 源代码
- 页面 HTML/CSS/JS
- 数据库结构的原始实现
- 图片和图标
- 商标、Logo、名称
- 文案中的品牌内容
- 受保护的素材和下载文件

## 实现方式

1. 观察参考系统功能。
2. 整理成自己的产品需求文档。
3. 用自己的数据库表、接口、页面和小程序重新实现。
4. 使用自己的品牌、UI 风格和代码结构。
5. 界面参考参考系统的 ERP 操作习惯，但不复制 HTML/CSS/JS、图片、图标和品牌素材。

## 最高优先级规则

- 所有模块都先参考用户提供的制造业版系统，不自行凭经验设计。
- 每个模块必须先采集参考系统的菜单、页面布局、筛选条件、表格列、按钮、弹窗/表单字段、状态、操作流程和报表口径。
- 没有采集到参考系统页面前，只能写准备性代码或草稿，不提交为正式实现。
- 可以提出更适合 yize-erp 的优化方案，但必须说明优化原因；用户未确认前，默认仍按参考系统实现。
- 优化只允许优化交互效率、单公司适配、技术可靠性和后续可维护性，不能随意删减参考系统已有功能。
