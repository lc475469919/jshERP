# 财务模块规格

## 参考范围

来源：参考系统制造业版菜单、`docs/reference-clean-room.md`、`docs/reference-modules-loafish.md`、`docs/specs/reference-shell-and-login.md`、`docs/specs/module-task-board.md` 与 `/tmp/loafish-menu-tree.txt` 中“财务”部分。

clean-room 约束：本规格只记录可参考的菜单结构、业务入口、参考路由、已出现的操作线索和后续采集项，不复制参考系统源码、数据库结构、页面代码、图片、品牌或受保护素材。

当前采集程度：已确认财务模块菜单、二级分组、页面入口和参考路由；尚未逐页采集真实页面的筛选条件、表格列、按钮、弹窗/表单字段、状态枚举、审批流程、计算公式和报表口径。因此本文只作为财务模块后续逐页采集的菜单级规格。

## 模块定位

财务模块位于一级菜单“财务”，覆盖业务单据产生后的往来账务处理、现金/银行等资金业务、费用报销、结算账户及资产等基础财务信息，以及客户、供应商、应收、应付、利润、费用和日常收支报表。

yize-erp 后续实现时应按参考系统菜单范围完整呈现入口；正式开发具体页面前，必须先逐页采集页面细节，再补充为正式规格。当前不进入数据库、接口、管理端页面、小程序或报表实现。

## 后续逐页采集项

### 列表页通用采集

每个财务业务列表至少需要确认：

- 页面标题、面包屑和页签名称。
- 顶部筛选条件、控件类型、默认值、查询按钮、重置按钮。
- 顶部按钮、批量按钮、行操作按钮。
- 表格列、列顺序、固定列、状态标签、金额格式、合计行、分页规则、排序规则。
- 新增、修改、删除、审核、反审核、提交、驳回、作废、关闭、导入、导出、打印、刷新等操作是否存在。
- 按钮在不同审核状态、单据状态、付款/收款状态下的启用或禁用规则。
- 来源单据、生成单据和原单跳转关系。
- 操作后对应收、应付、预收、预付、结算账户余额、资金流水、客户/供应商对账报表的影响。

### 表单页通用采集

每个财务业务表单至少需要确认：

- 主表字段、明细字段、字段分组、默认值、必填规则、校验规则。
- 客户、供应商、员工、部门、项目、结算账户、收支类别、费用类别等选择器来源。
- 金额、折扣、手续费、核销金额、未核销金额、余额等计算公式。
- 附件、图片、备注、审批备注、打印模板、操作日志是否存在。
- 单据编号规则、编号生成时机和编号是否允许手工修改。
- 审核、反审核、删除、作废、关闭后的数据影响和限制条件。

### 往来账务逐页采集

- 应付账款：供应商维度、单据来源、应付金额、已付金额、未付金额、账期、逾期、核销和明细钻取规则。
- 付款单：付款对象、付款来源、结算账户、付款明细、核销采购/其他应付规则、`shzt` 与 `ddzt` 枚举。
- 应收账款：客户维度、单据来源、应收金额、已收金额、未收金额、账期、逾期、核销和明细钻取规则。
- 收款单：收款对象、收款来源、结算账户、收款明细、核销销售/其他应收规则、`shzt` 与 `ddzt` 枚举。
- 其他应付单、其他应收单：业务类别、往来对象、金额方向、审核流程、是否进入应付/应收余额。
- 应收抵应付：抵扣对象选择、客户/供应商关系、可抵扣余额、抵扣金额校验、审核后余额变动。

### 资金业务逐页采集

- 日常支出、日常收入：收支类别、结算账户、经办人、项目/部门、附件、审核和资金流水生成规则。
- 预收款、预收退款：客户、结算账户、预收余额、退款条件、核销关系和报表影响。
- 预付款、预付退款：供应商、结算账户、预付余额、退款条件、核销关系和报表影响。
- 借入款、借出款：往来对象、借款日期、到期日期、还款状态、利息字段是否存在、资金流水影响。

### 费用报销逐页采集

- 报销申请：申请人、部门、费用类别、报销明细、金额合计、附件、提交和撤回规则。
- 报销审批：审批状态、审批人、通过、驳回、反审核、付款或生成资金流水的触发规则。
- 费用报销统计：统计维度、费用类别、部门/员工/项目口径、已报销/待报销/已付款金额口径。

### 其他信息逐页采集

- 结算账户：账户名称、账户类型、开户行、账号、期初余额、当前余额、启停用、默认账户规则。
- 账户互转：转出账户、转入账户、转账金额、手续费、转账日期、审核状态和流水生成规则。
- 固定资产：资产编号、名称、类别、原值、净值、购入日期、折旧、处置、打印、导入导出规则。
- 支出类别、收入类别：类别名称、编码、排序、启停用、是否系统预置、被引用后是否允许删除。
- 资金流水：来源单据、来源模块、发生日期、收入金额、支出金额、账户余额、原单跳转和导出规则。

### 财务报表逐页采集

- 客户对账统计、客户对账明细：查询条件、客户分组、期初、本期应收、本期收款、本期退货/抵扣、期末余额和明细钻取规则。
- 供应商对账统计、供应商对账明细：查询条件、供应商分组、期初、本期应付、本期付款、本期退货/抵扣、期末余额和明细钻取规则。
- 新旧客户/供应商对账统计：页面差异、字段差异、是否都需要保留、是否存在新版替代旧版的提示。
- 未收账款统计、未付账款统计：未收/未付口径、账期、逾期天数、单据来源、客户/供应商汇总和明细钻取。
- 利润统计：收入、成本、费用、其他收支、退货、税费等指标和计算公式。
- 日常收支统计：收入类别、支出类别、账户、部门/项目、日期范围、合计和明细钻取。
- 所有报表需确认是否支持导出、打印、钻取原单、合计行、分组小计和权限控制。

## 依赖模块

后续正式开发财务模块前，需要至少完成或确认以下基础能力：

- 客户信息、供应商管理：用于应收、应付、收款、付款和对账报表。
- 员工、部门、项目：用于经办人、申请人、审批人、费用归属和项目统计。
- 商品、采购、销售、库管单据：用于应收应付来源、成本和利润统计。
- 结算账户、收入类别、支出类别：用于资金业务和资金流水。
- 单据编号规则：用于付款单、收款单、日常收支、预收预付、借款、报销、账户互转等单据。
- 权限、审批、操作日志和附件：用于审核留痕、审批流、附件凭证和操作追溯。

## 当前结论

当前阶段只完成财务模块菜单级 clean-room 规格整理。下一步如进入财务实现，必须先按页面逐项采集参考系统的筛选、表格、按钮、表单、状态、流程、计算公式和报表口径，再补充正式规格。
