# 模块并行任务板

## 规则

- 一个模块一个任务。
- 每个模块先整理 clean-room 规格，再进入接口、数据库、页面开发。
- 规格文档只记录参考系统已采集到的菜单、页面、筛选、表格、按钮、弹窗、状态和流程。
- 未采集到页面细节的功能，只能标记为“待采集”，不能当作正式实现。
- 多任务同步推进时，每个任务只改自己的规格文件或明确分配的代码范围。

## 开发顺序

1. 先把主框架菜单全部接成参考入口。
2. 基础资料继续补齐已采集页面。
3. 生产、采购、销售、库管进入单据前，先完成共用基础：客户、供应商、商品、仓库、员工、部门、结算账户。
4. 单据模块按“列表 -> 新增/修改 -> 审核/反审核 -> 打印/导入/导出 -> 报表”的顺序开发。
