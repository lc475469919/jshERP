# 系统底座规格

## 基本信息

- 阶段：第 0 阶段
- 目标：先建立登录、用户、角色、权限、菜单、字典、单据编号规则，为后续基础资料和业务单据提供统一底座。
- 参考来源：参考系统的“设置”“基础”“人事”模块中可见的系统配置、基础信息、员工薪资相关入口。
- 设计原则：单公司模式，不做租户，不出现 `tenant_id`。

## 菜单规划

### 系统管理

- 用户管理
- 角色管理
- 菜单管理
- 权限分配
- 字典管理
- 单据编号规则
- 操作日志

### 基础配置

- 标记设置
- 商品属性
- 物流公司

说明：参考系统把部分配置分散在“设置”“基础”“物流”等模块里。yize-erp 先统一放入系统底座和基础配置，后续业务模块成熟后再按实际使用频率调整菜单。

## 登录

- 管理端使用账号密码登录。
- 登录成功返回访问令牌、用户信息、菜单树、权限码。
- 退出登录使当前令牌失效。
- 当前阶段先支持管理端登录；小程序登录在移动端阶段补齐。

## 用户管理

### 列表页

- 查询条件：姓名、手机号、账号、状态、角色。
- 表格列：账号、姓名、手机号、角色、状态、最后登录时间、创建时间。
- 操作：新增、编辑、启用、停用、重置密码、删除。

### 表单字段

- 账号
- 姓名
- 手机号
- 密码
- 角色
- 状态
- 备注

## 角色和权限

- 角色用于绑定菜单和操作权限。
- 权限粒度先覆盖菜单访问和常用按钮：查看、新增、编辑、删除、审核、导入、导出、打印。
- 超级管理员拥有全部权限。
- 角色删除前校验是否被用户使用。

## 菜单管理

- 菜单支持目录、页面、按钮三类。
- 菜单树用于管理端侧边栏。
- 按钮权限用于页面操作控制。
- 菜单字段：名称、类型、父级、路由、组件、权限码、图标、排序、显示状态、启用状态。

## 字典管理

- 字典用于单据状态、启停状态、性别、计薪方式、单位类型等可配置选项。
- 字典由字典类型和字典项组成。
- 字典项字段：标签、值、排序、颜色、状态、备注。

## 单据编号规则

- 用于后续采购、销售、库存、生产、财务、工资等单据生成编号。
- 规则字段：单据类型、前缀、日期格式、流水位数、当前流水号、是否按日重置、状态。
- 示例：生产任务 `MO-{yyyyMMdd}-{0001}`，采购订单 `PO-{yyyyMMdd}-{0001}`。

## 操作日志

- 记录用户、模块、操作类型、业务编号、请求结果、操作时间。
- 第 0 阶段先做数据结构和基础记录能力；复杂审计报表后续补齐。

## 验收标准

- 管理端可以登录并加载当前用户菜单。
- 可以维护用户、角色、菜单、字典、编号规则。
- 权限可以控制页面入口和按钮显示。
- 后端具备 Flyway 表结构、实体、接口、基础校验。
- 不出现租户字段和多租户逻辑。
