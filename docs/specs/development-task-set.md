# 开发任务体系

## 总规则

- 一个模块一个任务。
- 一个页面一个采集记录。
- 每次开发只做一个清晰任务包，不把未采集页面当成已完成页面。
- 每个任务包结束必须验证、提交、推送到 GitHub。

## 任务状态

- 待采集：只有菜单和参考路径，还没有页面细节。
- 已采集：已记录筛选、表格、按钮、弹窗、状态和流程。
- 开发中：正在做数据库、接口、管理端页面或小程序。
- 已验证：构建或后端打包通过。
- 已备份：已提交并推送到 GitHub。
