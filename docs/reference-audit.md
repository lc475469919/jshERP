# 参考系统一致性审计

审计时间：2026-04-26

目标：明确当前代码里哪些内容已经按参考系统采集后实现，哪些只是为了工程可运行先搭建的自有实现，避免后续把临时实现误认为正式方案。

## 结论

当前项目不是完全按参考系统复刻完成的状态。

- 已经按参考系统采集并实现第一版的是“基础”模块第一批：商品信息、商品属性、供应商管理、项目信息、物流公司。
- 系统底座、登录页、首页仪表盘、系统管理页面、菜单占位项，是为了让新项目能登录、能维护权限、能继续开发而先写的自有实现。
- 基础资料第一版虽然来自参考系统，但仍有删减和占位，不能视为已经完整达到参考系统。

## 部分参考但仍有自有简化

### 管理端基础资料页面

位置：`apps/admin-web/src/views/masterdata/MasterDataView.vue`

说明：页面框架参考了参考系统的左树、筛选、工具栏、表格、Tab 等结构，但当前把多个参考页面收敛到一个通用 Vue 页面中，且部分按钮是禁用占位。

处理方式：后续每个页面需要继续按参考系统补齐列、按钮、弹窗、批量操作、导入导出、状态流转。

### 后端基础资料模型

位置：`apps/backend/src/main/java/com/yize/erp/masterdata`

说明：实体和接口是按参考系统第一批页面重新设计的原创实现，但字段还没有覆盖参考系统全部页面能力。

处理方式：以后补字段时必须先更新 `docs/specs/master-data.md`，再改数据库迁移、实体、接口和页面。

## 自有实现，未逐页参考

### 登录页

位置：`apps/admin-web/src/views/LoginView.vue`

当前状态：自有设计，只参考了 ERP 登录的基本账号密码模式，没有逐项复刻参考系统登录页。

参考系统登录页仍需采集：

- 背景视频/背景区域
- 账号登录和扫码登录切换
- 版本选择
- 记住密码
- 忘记密码
- 企业微信相关入口

处理方式：登录页后续要按参考系统重新采集并重做交互，但使用 yize-erp 自有品牌和素材。

### 菜单展示和占位菜单

位置：`apps/admin-web/src/layouts/BasicLayout.vue`

当前状态：顶部/侧边菜单渲染是自有工程实现；其中禁用占位菜单只是根据已采集的一级模块临时摆放，不是正式功能。

处理方式：占位菜单后续要从后端菜单种子和参考系统模块清单生成，不能长期写死在前端。

### 首页仪表盘

位置：`apps/admin-web/src/views/dashboard/DashboardView.vue`

当前状态：自有示例页面，不是参考系统生产看板或首页。

处理方式：在采集参考系统首页或生产看板前，不把当前仪表盘作为正式功能。后续应按参考系统的首页/看板字段、指标、筛选和图表口径重做。

### 系统管理

位置：

- `apps/admin-web/src/views/system/SystemListView.vue`
- `apps/admin-web/src/views/system/MenuManageView.vue`

处理方式：可以保留作为技术底座，但设置/参数类页面以后仍要按参考系统采集后调整菜单位置、字段和操作方式。

### 小程序骨架

位置：`apps/miniprogram`

当前状态：自有最小登录和首页骨架，尚未按参考系统移动端或真实业务流程采集。

处理方式：进入移动端阶段前，先确定参考系统是否有移动端页面；没有可用参考时，再按管理端业务流程做移动端适配方案，并单独记录为优化。

## 文档中需要纠偏的内容

- `docs/current-plan.md` 里“商品分类、计量单位、商品/物料档案、客户、仓库、部门、员工、账户”的旧顺序带有自有规划色彩，需要改成先按参考系统“基础”模块菜单继续补齐。
- `docs/specs/master-data.md` 的“建议优化”只能作为建议，用户未确认前不能用它删除参考系统字段或功能。
- `docs/specs/system-foundation.md` 需要标注系统底座属于工程必要能力，不等于已完整参考参考系统设置模块。

## 后续执行规则

- 每个页面先采集参考系统，再开发。
- 临时自有实现必须在本文档里标注，不得混入正式验收标准。
- 已有自有实现可以保留支撑开发，但后续要逐项用参考系统规格替换。
- 没有参考采集的模块，不进入正式开发。
