# 我的 ERP 系统蓝图

## 目标

基于当前 `jshERP` 项目打造一套属于自己的 ERP 系统，包含电脑端管理后台、后端 API、微信小程序移动端，并参考 `ERPNext/Frappe` 的成熟业务流程补齐功能设计。

生产模块按 `docs/manufacturing-reference-backlog.md` 对标骆飞智能云 ERP 制造业版推进：不受当前轻量生产模块限制，按参考系统的生产、委外、工序、工资和生产报表能力重新实现一套自己的制造业版 ERP。

## 结合方式

两套系统不直接合并源码。

- `jshERP` 作为主工程：继续使用现有 Spring Boot、MyBatis、Vue 2、微信小程序结构开发。
- `ERPNext` 作为参考系统：参考它的模块划分、单据流转、字段设计、权限和报表思路。
- 数据和功能逐步迁移：把适合自己的流程重新设计到 jshERP 的数据库、接口、后台页面和小程序页面中。

这样做可以避免两套技术栈强行混合带来的维护成本，也能保留 jshERP 已经完成的进销存、审批、生产、小程序基础。

## 产品定位

第一阶段定位为适合中小企业使用的轻量 ERP：

- 进销存一体化
- 客户、供应商、商品、仓库、员工等基础资料
- 销售、采购、库存、财务、审批
- 生产管理和委外管理
- 员工、考勤、计件工资、工资核算
- 老板/员工微信小程序移动办公
- 单公司本地化部署，不做 SaaS 平台

## 单公司模式

系统按自己公司内部使用来设计，不提供外部企业注册、公司切换、套餐、到期续费、平台运营后台等功能。

保留：

- 公司资料
- 部门
- 员工/用户
- 角色权限
- 仓库、门店、生产车间
- 单据审批流程
- 数据备份和系统配置

移除或隐藏：

- 外部企业注册
- 公司切换
- 套餐/版本售卖
- 到期时间
- token 后缀隔离
- 面向云平台运营方的管理页面

数据库层面可以暂时保留历史兼容字段，但新功能一律围绕“公司内部管理”设计。后续等功能稳定后，再做一次专门的数据库清理迁移。

## 模块规划

### 基础资料

- 商品资料、商品分类、计量单位、条码、规格型号
- 客户、供应商、联系人
- 仓库、库位
- 经手人、员工、部门、角色权限
- 账户、收支项目
- 工序、不良品项、BOM 模板

### 采购管理

- 采购申请
- 采购订单
- 采购入库
- 采购退货
- 供应商对账
- 应付款

### 销售管理

- 销售报价
- 销售订单
- 销售出库
- 销售退货
- 客户对账
- 应收款

### 库存管理

- 库存查询
- 调拨
- 盘点
- 其他入库、其他出库
- 库存预警
- 批次/序列号作为后续增强

### 财务管理

- 收款、付款
- 账户流水
- 应收应付
- 利润统计
- 费用登记

### 员工与考勤

- 员工档案
- 部门/岗位
- 上下班打卡
- 外勤打卡
- 请假、调休、加班
- 考勤月报
- 迟到、早退、缺勤统计
- 微信小程序打卡

### 工资管理

- 工资项目设置
- 员工基础工资
- 计时工资
- 计件工资
- 工序计件单价
- 加班工资
- 请假/缺勤扣款
- 社保、公积金、个税等扣款项预留
- 工资单生成
- 工资发放记录
- 员工小程序查看工资条

工资计算联动：

- 工序汇报记录可生成计件工资
- 考勤记录可生成计时工资、加班工资和缺勤扣款
- 生产质检的不良/报废数量可参与计件工资扣减规则
- 工资单确认后生成财务付款或工资发放记录
- 工资单锁定后不允许随意修改来源考勤和计件数据

### 审批中心

- 销售订单审批
- 采购订单审批
- 付款审批
- 生产任务审批
- 小程序移动审批

### 生产管理

自制业务：

- BOM 管理
- 生产任务
- 生产领料
- 用料登记
- 工序汇报
- 生产质检
- 成品入库

委外业务：

- 委外任务
- 物料出库
- 委外质检
- 委外成品入库
- 委外付款情况

生产报表：

- 生产看板
- 生产进度跟踪
- 订单进度跟踪
- 物料追踪
- 缺料分析
- 生产成本统计
- 计件工资统计
- 工序工资统计

### 微信小程序

第一批移动端能力：

- 登录和个人中心
- 首页经营概览
- 扫码查商品/库存
- 客户、供应商、商品查询
- 销售出库、采购入库查询和录入
- 审批中心
- 库存预警

第二批移动端能力：

- 移动开销售订单
- 移动开采购订单
- 工序汇报
- 生产质检
- 生产任务进度查看
- 仓库扫码出入库
- 上下班打卡
- 外勤打卡
- 请假/加班申请
- 查看考勤
- 查看工资条

## 技术路线

主工程继续沿用当前技术栈：

- 后端：Spring Boot + MyBatis + MySQL + Redis
- 管理端：Vue 2 + Ant Design Vue
- 小程序：原生微信小程序
- 本地运行：项目已有 `scripts/start-local.sh` 和 `scripts/stop-local.sh`

ERPNext 只作为参考来源：

- 参考模块菜单
- 参考单据生命周期
- 参考字段和报表
- 参考制造业流程
- 不引入 ERPNext 的 Python/Frappe 运行时到 jshERP 主工程

## 开发优先级

### 第 1 阶段：打稳基础

- 跑通本地后端、前端、小程序
- 整理单公司登录、权限、菜单
- 清理或隐藏云平台相关入口
- 确认数据库初始化和本地启动脚本稳定
- 梳理已有未提交改动

### 第 2 阶段：进销存和小程序闭环

- 商品、客户、供应商移动端查询
- 销售出库、采购入库移动端录入
- 库存查询和库存预警
- 审批移动端可用

### 第 3 阶段：生产管理闭环

- BOM 到生产任务
- 生产领料到用料登记
- 工序汇报到生产质检
- 成品入库自动回写生产任务状态

### 第 4 阶段：委外和成本

- 委外任务
- 委外出库、质检、入库
- 委外付款
- 生产成本统计
- 工序计件工资统计

### 第 5 阶段：员工、考勤和工资

- 员工档案
- 打卡考勤
- 请假、加班、调休
- 计时工资
- 计件工资
- 工资单生成
- 小程序打卡和工资条

### 第 6 阶段：报表和经营看板

- 老板看板
- 销售、采购、库存、财务报表
- 生产进度、缺料、成本、工资报表
- 小程序经营概览增强

## 近期可执行任务

1. 先确认系统名称、Logo、登录页品牌。
2. 清理当前 git 未提交改动，避免后续开发混乱。
3. 启动本地 jshERP，验证后台和小程序登录。
4. 选一个核心业务闭环先做完，建议从“小程序销售出库 + 审批 + 库存扣减”开始。
5. 再参考 ERPNext 的生产制造流程，继续完善生产模块。
