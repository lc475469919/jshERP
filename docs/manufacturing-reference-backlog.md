## 原则

参考系统制造业版中已经具备的生产能力，全部纳入我们自己的单公司 ERP 路线图。

实现方式改为“对标重做”：

- 不以当前轻量生产模块为限制。
- 不直接复制对方源码、图片、商标、数据库或受保护素材。
- 按参考系统的菜单、流程、字段、状态、公式、报表和交互方式，重新实现一套属于自己的 ERP。
- 当前 `jshERP` 只作为技术底座和可复用基础能力，生产模块可以推倒重建。
- 旧轻量生产模块后续可以删除或废弃，由新的制造业版生产模块替代。

## 关键生产规则

### 批量生产领料

系统以 BOM 物料清单为依据，自动计算本次领料数。

公式：

```text
领料数 = 所需数量 - 已领料数 - 已退数 - 废料数
```

数量小于等于 0 的物料忽略。

### 批量用料登记

系统以 BOM 物料清单为依据，自动计算本次用料数。

公式：

```text
用料数 = 已领料数 - 已退数 - 废料数
```

数量小于等于 0 的物料忽略。

### 批量生产质检

系统以计划生产数和已验收数为依据，自动计算本次数量。

公式：

```text
本次验收数 = 计划生产数 - 累计已验收数
```

### 批量成品入库

启用生产验收时：

```text
本次入库数 = 累计验收合格数 - 累计已入库数
```

停用生产验收时：

```text
本次入库数 = 计划生产数 - 累计已入库数
```

成品入库支持：

- 选择入库仓库
- 倒冲领料
- 标记生产完工
- 选择倒冲领料仓库

### 批量完工

系统提示需要人工确认以下情况：

- 领料状况
- 工序执行状况
- 验收状况
- 入库状况

## 缺料追踪字段参考

主表字段：

- 物料编号

- 规格
- 颜色
- 所需数量
- 可用库存
- 待入库数
- 缺料数
- 订购中

缺料公式：

```text
缺料数 = 所需数量 - 可用库存数 - 待入库数 - 领料数 - 退料数 - 废料数
```

订购中：

- 已生成采购订单中，包含未审核订单，但未生成采购单的数量。

- 生产编号
